# The labs

This book develops the physics of glaciers in words and equations, and it turns that physics into computation through a set of labs: self-contained exercises that you run yourself, change, and learn from by experiment. The aim is to learn how ice behaves by making a model do it, not only by reading about it.

The labs are distributed through the book rather than collected at the end. Each one sits with the chapter whose physics it brings to life, and points back to that chapter for the theory, so the labs and the main text are read together. A lab on the flow law follows the rheology chapter, a lab on the friction laws follows the chapter on sliding, a lab on the dielectric spectrum follows the chapter on electromagnetic waves, and so on.

They come in two kinds. The lighter labs need only the standard scientific Python stack, numpy and matplotlib, and reproduce an analytical result or plot a relationship from the text, such as Glen's creep curve, the Vialov profile, or the effective pressure in a channel against a cavity. The heavier labs build a working glacier model with [icepack](https://icepack.github.io/), which runs on the finite-element library [Firedrake](https://www.firedrakeproject.org/), and these solve the diagnostic and prognostic flow problems on real geometries. A few labs use other group tools, such as the ApRES processing library or the specfab fabric model.

All of the labs are written as Jupyter notebooks and are meant to be run in the book's Docker image, which provides icepack, Firedrake, and the other dependencies already installed, with a kernel named "Firedrake (icepack)". The setup is shown in {doc}`running-icepack-docker`. Because that stack is too heavy to run on a documentation server, the labs are not executed when the book is built: what you see on the page is the code and an explanation of what it produces, and to see the output and do the exercises you run the notebook yourself in the container.

Most of the labs are built in two stages that mirror the structure of the book itself: a diagnostic stage first, then a prognostic one. The diagnostic stage solves for the velocity of a glacier whose shape is fixed, which is the problem the front half of the book prepares, and a diagnostic lab can therefore stand on its own beside the chapter it illustrates. The prognostic stage takes that velocity and steps the geometry forward in time under a mass balance and a climate forcing, the problem of the later chapters. Wherever it is natural, a lab first sets up and solves the diagnostic problem and then evolves it, so the reader sees the same model used both ways.

Following that arc, the icepack labs are ordered so that each adds a single piece of physics to a model already met. The first builds a floating ice shelf and solves for its velocity, the diagnostic problem of the book. Later labs add basal sliding, couple the flow to temperature, and step the geometry forward in time, so that by the end you have assembled a prognostic model that contains most of the processes covered in the main text.
